### 一、 前端与交互技术

1. **DOM（文档对象模型）的本质**
    - DOM 是浏览器将 HTML 文档解析后在内存中生成的“树形结构”。我们通过 JavaScript 操作的（如 `appendChild`、`classList.add`）都是这棵内存中的 DOM 树，而不是直接修改磁盘上的 HTML 源文件。
2. **动态表单与元素创建**
    - `document.createElement("input")` 用于在内存中动态创建节点，配合 `type='hidden'` 和 `name/value` 属性，它可以作为“数据容器”偷偷将参数塞进表单提交给后端。
    - `document.body.appendChild(form)` 是将内存中的节点挂载到页面 DOM 树中，这是 `form.submit()` 能够成功提交的前提。
3. **事件监听与交互逻辑**
    - **回车提交**：通过监听 `keydown` 事件并判断 `e.key === 'Enter'` 实现。
    - **遮罩层关闭弹窗**：利用事件冒泡机制，通过判断 `e.target === this` 确保只有点击遮罩层本身（而非内部输入框）才关闭弹窗。
4. **请求转发与重定向的区别**
    - **转发** (`forward`)：是服务器内部行为，一次请求，地址栏不变。因为 `request` 对象是同一个，所以可以通过 `setAttribute` 传递错误信息，且表单数据会保留（回显），页面不会完全重置刷新。
    - **重定向** (`sendRedirect`)：两次请求，地址栏改变，无法通过 request 传值，页面会重置为空白状态。

### 二、 Java 面向对象核心思想

1. **多态与动态绑定（面向接口编程的底层原理）**
    - `UserDAO user = new UserDAOImpl();` 体现了多态。
    - **编译期看左边**：变量声明的类型决定了能调用哪些方法（只能调接口定义的）。
    - **运行期看右边**：堆内存中实际创建的对象类型决定了具体执行哪个类的逻辑（动态绑定）。
2. **依赖注入与解耦的优势**
    - 直接 `new` 实现类会导致代码强耦合。如果实现类换成 `Impl12`，必须修改调用方的源代码。
    - 现代开发使用依赖注入（如 Spring 的 `@Autowired`），由容器管理对象的生命周期。此时如果替换实现类，调用方代码可以做到零修改，这就是面向接口编程带来的解耦优势。

### 三、 后端 DAO 层与数据库交互

1. **DAO 模式的架构规范**
    - **接口** (`UserDAO.java`)：定义业务所需的抽象方法契约（如增删改查）。
    - **实现类** (`UserDAOImpl.java`)：编写具体的 JDBC 和 SQL 逻辑，负责与数据库交互。IDE 中通常用绿色图标标识实现类。
2. **核心方法的运行机制**
    - **`login` 方法**：执行 `SELECT * FROM user WHERE username = ? AND password = ?`，查到则封装为 User 对象返回，否则返回 null。业务层通过判断是否为 null 决定登录成功或失败。
    - **`checkUsernameExists` 方法**：执行 `SELECT COUNT(*) FROM user WHERE username = ?`，只返回统计数字。这是一种性能优化，无需查询和封装完整的用户数据即可判断存在性。
    - **方法独立性**：`login` 和 `check` 在代码逻辑上是完全独立的。`check` 是否被调用，完全取决于业务层（如注册时防重复）是否主动调用它，`login` 内部不会自动调用 `check`。
3. **JDBC 操作模板**
    - 遵循：获取连接 -> 预处理 SQL（防注入） -> 设置参数 -> 执行更新/查询 -> 处理结果集 -> 关闭资源（finally 块中）的标准流程。

### 四、 Web 前后端交互核心（重点突破）
1. **隐藏域 `<input type="hidden">`**：懂得了它在页面上不可见，但能悄摸摸把 `action=add` 这种指令传给后端，用于同一个 Servlet 处理多种业务逻辑。
2. **重定向 vs 转发（今天的灵魂考点）**：
    - **重定向**（`sendRedirect`）：两次请求，地址栏变，参数丢失。**适用于增删改后的跳转**，防止表单重复提交（PRG 模式）。
    - **转发**（`forward`）：一次请求，地址栏不变，参数保留。**适用于查询后的页面跳转**，能携带 Request 域数据。
3. **`getParameter` vs `getAttribute`**：
    - `getParameter()`：只能拿字符串（来自前端表单或 URL）。
    - `getAttribute()`：拿 Java 对象（来自后端 `setAttribute`），必须强转类型。你成功避开了这个大坑！
4. **图片在项目中的流转**：理清了“图片存本地磁盘，路径存数据库 URL”的架构，知道了前端拿 URL 当 `src`，服务器需配置静态资源映射。
